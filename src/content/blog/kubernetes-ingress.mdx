The MongoDB lab exposed Mongo Express at something like `http://<node-ip>:30000`. That works in a lab. Production apps expect `https://app.example.com`.

**Ingress** sits in front of internal services and routes HTTP(S) by hostname and path. No NodePort in the URL.

## NodePort vs Ingress

**External service (LoadBalancer + NodePort):**

```yaml
apiVersion: v1
kind: Service
metadata:
  name: mongo-express-service
spec:
  selector:
    app: mongo-express
  type: LoadBalancer
  ports:
    - protocol: TCP
      port: 8081
      targetPort: 8081
      nodePort: 30000
```

Access: `http://<node-ip>:30000`

**Ingress approach:** internal ClusterIP service + Ingress rule mapping a hostname to that service.

<figure style="text-align:center; margin: 1.25rem 0;" class="blog-figure-md">
  <img src="/resources/kubernetes/c3f7195e680642c38c4125bbbd87911b.png" alt="Ingress routing host and path to internal service" style="display:block; margin:0 auto; max-width:100%; height:auto;" />
</figure>

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: myapp-ingress
spec:
  rules:
  - host: myapp.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: myapp-internal-service
            port:
              number: 80
```

- `host`: the domain clients use.
- `paths`: URL path prefix forwarded to the backend service.
- The `http` key is the protocol block, not a literal URL.

<figure style="text-align:center; margin: 1.25rem 0;" class="blog-figure-wide">
  <img src="/resources/kubernetes/aa3a6e1dceeb42ff91f36129691d8922.png" alt="Full ingress and internal service flow" style="display:block; margin:0 auto; max-width:100%; height:auto;" />
</figure>

The hostname must resolve to the cluster entrypoint (load balancer IP or node IP). On Minikube you fake this with `/etc/hosts`.

## Ingress controller

Ingress rules are just config. Something has to enforce them: the **ingress controller**. Usually a pod (nginx, traefik, etc.) that watches Ingress objects and updates its proxy config.

<figure style="text-align:center; margin: 1.25rem 0;" class="blog-figure-md">
  <img src="/resources/kubernetes/0501bfd2339749afbe116adff2ce69a5.png" alt="Ingress controller evaluating rules and forwarding traffic" style="display:block; margin:0 auto; max-width:100%; height:auto;" />
</figure>

- **Cloud**: managed LB often forwards to the ingress controller automatically.
- **Bare metal**: you run your own reverse proxy with a public IP pointing at the controller.

### Example: kubernetes-dashboard

Minikube ships a dashboard with an internal service:

<figure style="text-align:center; margin: 1.25rem 0;" class="blog-figure-wide">
  <img src="/resources/kubernetes/414c03c51c3a4d618158e6d94da98fce.png" alt="kubernetes-dashboard internal service" style="display:block; margin:0 auto; max-width:100%; height:auto;" />
</figure>

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: dashboard-ingress
  namespace: kubernetes-dashboard
  annotations:
    kubernetes.io/ingress.class: "nginx"
spec:
  rules:
  - host: dashboard.com
    http:
      paths:
      - path: /
        pathType: Exact
        backend:
          service:
            name: kubernetes-dashboard
            port:
              number: 80
```

```bash
kubectl apply -f dashboard-ingress.yaml
kubectl get ingress -n kubernetes-dashboard
```

<figure style="text-align:center; margin: 1.25rem 0;" class="blog-figure-wide">
  <img src="/resources/kubernetes/ff454ea4c9c64359b12cc1123bf1b730.png" alt="ingress assigned an address" style="display:block; margin:0 auto; max-width:100%; height:auto;" />
</figure>

Map the host locally:

```bash
# Linux / macOS: add to /etc/hosts
<ingress-ip>  dashboard.com
```

Then browse to `http://dashboard.com`.

## TLS

Terminate HTTPS at the ingress with a TLS Secret:

<figure style="text-align:center; margin: 1.25rem 0;" class="blog-figure-wide">
  <img src="/resources/kubernetes/42492e0b98c8419b81849a1e9eca004a.png" alt="TLS certificate configuration for ingress" style="display:block; margin:0 auto; max-width:100%; height:auto;" />
</figure>

Rules:

1. Secret keys must be `tls.crt` and `tls.key`.
2. Values are **file contents**, not file paths.
3. Secret and Ingress must be in the **same namespace**.

```bash
kubectl create secret tls my-tls-secret \
  --cert=path/to/tls.crt \
  --key=path/to/tls.key \
  -n my-namespace
```

Reference it in the Ingress `spec.tls` block.
